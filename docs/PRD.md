# 小红书需求雷达 - 产品需求文档

## 一、产品本质

这是一个个人用的小红书"需求雷达"工具，目的是从新榜数据中，识别"被低估但可变现的关键词"。

## 二、核心问题

当前问题：新榜只能看到「什么火了」，但看不到：
- 用户为什么收藏
- 真实需求是什么
- 能不能变现

👉 所以需要一个系统，把：**内容数据 → 用户需求 → 产品机会**

## 三、核心能力

### 1️⃣ 过滤"无效流量"

目标：从榜单中剔除娱乐内容，只保留"有需求的内容"

判断逻辑：
- 收藏高 → 有需求
- 评论中有"求方法" → 强需求

输出：标记哪些内容是"值得分析的"

### 2️⃣ 提取"真实关键词"

目标：把标题转成"用户实际会搜索的关键词"

例如：
- 原标题：复古茶会
- 转换为：茶会布置、茶会流程、茶会策划

本质：从"内容表达" → "搜索意图"

### 3️⃣ 判断"是否被低估"

目标：找出还没被卷，但已经有需求的关键词

判断维度：
- 收藏比例高（说明有需求）
- 内容不多（说明竞争低）
- 评论有"求资源"（说明未满足）

输出：一个"机会评分"

### 4️⃣ 给出"变现路径"

目标：每个关键词都必须能回答：这个需求可以卖什么？

输出必须包含：
- 用户想解决什么问题
- 可以做什么产品（模板 / 服务 / 工具）

## 四、输入 / 输出

### 输入（Input）

来自新榜：
- 标题
- 点赞
- 收藏
- 评论数

### 输出（Output）

每一条必须包含：
1. **关键词**：转换后的需求关键词
2. **需求描述**：用户在解决什么问题
3. **机会评分**：是否值得做（高 / 中 / 低）
4. **产品建议**：可以卖什么

## 五、系统流程

```
Step 1: 输入榜单数据
    ↓
Step 2: 筛选（去掉娱乐内容，保留高收藏内容）
    ↓
Step 3: 提取关键词（标题 → 用户需求）
    ↓
Step 4: 判断机会（是否被低估）
    ↓
Step 5: 输出结果（关键词 + 需求 + 产品机会）
```

## 六、使用方式

用户每天做一件事：
1. 打开新榜
2. 复制榜单数据
3. 输入工具

工具输出：今日最值得做的 3–5 个关键词

## 七、成功标准

1. 是否能稳定输出：每天 3 个高质量关键词
2. 用户看到后是否会觉得："这个我可以做"
3. 是否能直接导向：内容选题 / 产品设计

## 八、边界

这个工具：
- ❌ 不做：热点榜单、数据统计、UI复杂系统
- ✅ 只做：决策建议

## 九、核心原则

目标不是分析数据，而是找到"可以赚钱的需求"。

优先输出：
- 有收藏
- 有未满足需求
- 有明确变现路径

避免输出：
- 娱乐内容
- 情绪内容
- 无法变现的热点

## 十、最终一句话总结

这个工具的本质不是"看趋势"，而是：**在信息流里找到"钱还没被赚走的需求"**
