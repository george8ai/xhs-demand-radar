# XHS Demand Radar - 小红书需求雷达

从新榜低粉爆文榜数据中识别"被低估但可变现的关键词"。

## 触发条件

- 用户说"分析小红书需求"、"跑需求雷达"、"今日需求分析"
- 收到定时任务触发

## 数据位置

```
~/Workspace/xhs-demand-radar/data/newrank_YYYY-MM-DD.json
```

## 分析流程

### Step 1: 加载数据

读取最新的 JSON 数据文件。

### Step 2: 过滤无效内容

剔除以下类型（无变现价值）：
- 纯娱乐/搞笑内容
- 明星八卦
- 情绪宣泄类
- 无法提取需求的内容

保留标准：
- 收藏数高（说明有需求）
- 分类为实用类（美食、家居、教育、母婴、穿搭等）

### Step 3: 提取需求关键词

将标题转换为"用户会搜索的关键词"：

示例：
- "复古茶会" → 茶会布置、茶会流程、茶会策划
- "大学生晚上的固定操作" → 大学生晚间routine、晚自习安排

### Step 4: 评估机会

对每个关键词评分（高/中/低）：

| 维度 | 高分标准 |
|------|---------|
| 需求强度 | 收藏 > 点赞（说明想保存） |
| 竞争程度 | 低粉能爆（说明内容少） |
| 变现潜力 | 能对应具体产品/服务 |

### Step 5: 输出结果

每个关键词必须包含：
1. **关键词**：转换后的需求关键词
2. **需求描述**：用户在解决什么问题
3. **机会评分**：⭐⭐⭐⭐⭐ (1-5星)
4. **产品建议**：可以卖什么

## 输出格式

```
📊 小红书需求雷达 (YYYY-MM-DD)

🔥 关键词: XXX
   需求: 用户想要...
   机会: ⭐⭐⭐⭐ (收藏高、内容少)
   变现: 模板/服务/工具

🔥 关键词: YYY
   ...
```

## 注意事项

- 每次输出 3-5 个最有价值的关键词
- 优先输出有明确变现路径的
- 避免输出无法落地的泛需求
